# FluxDeck README 面向使用者重写设计

## 目标

将仓库根目录 `README.md` 从简短的开发态说明，重写为面向使用者的项目首页文档，让首次进入仓库的人能快速理解：

- FluxDeck 是什么
- 当前已经可用到什么程度
- 适合哪些使用场景
- 如何最快跑起来
- 去哪里看更详细的使用与运维文档

## 当前问题

现有 README 偏向早期 MVP 阶段：

- 对当前项目能力覆盖不完整
- 没有突出“原生桌面端优先”的最新方向
- 没有明确说明 OpenAI / Anthropic 网关、Provider / Gateway 管理、运行监控等当前能力
- 快速开始路径较窄，更像开发者自用笔记

对于外部使用者来说，首页信息不足以建立正确预期。

## 目标读者

本次 README 的主目标读者是使用者，而不是仓库维护者或核心开发者。

这意味着 README 需要优先回答：

1. 这个项目能解决什么问题
2. 我现在能不能用
3. 我怎么最快跑起来
4. 哪些能力已经稳定，哪些还在演进

## 推荐结构

### 1. 项目简介

用 2 到 4 行说明：

- FluxDeck 是本地运行的 LLM Gateway / Provider 管理与流量观测工具
- 面向 macOS，本地优先
- 支持 OpenAI / Anthropic 兼容链路

### 2. 当前状态

明确当前阶段：

- 原生桌面端优先
- `fluxd` / `fluxctl` / 原生桌面端已经可用
- Web 桌面端保留但不是当前主线

### 3. 当前能力

用使用者语言描述当前可用功能：

- Provider 管理
- Gateway 管理
- OpenAI / Anthropic 入站
- 请求日志与基础流量指标
- 运行中 Gateway 更新后自动重启

### 4. 适用场景

列出 3 到 5 个典型场景，例如：

- 在本机统一管理多个上游 LLM Provider
- 给本地工具提供稳定网关入口
- 调试 Anthropic 兼容链路
- 观察请求日志、延迟和 token 使用

### 5. 快速开始

聚焦最短路径：

- 启动 `fluxd`
- 用 `fluxctl` 创建 Provider / Gateway
- 打开原生桌面端或 Web 开发端

### 6. 关键文档入口

保留文档索引，但从“文档目录”改为“下一步该看什么”。

### 7. 验证命令

保留提交前验证命令，方便高级使用者和维护者。

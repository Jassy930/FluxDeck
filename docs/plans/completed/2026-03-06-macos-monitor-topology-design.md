# FluxDeck macOS 监控台与路由拓扑设计

## 目标

将 `apps/desktop` 从当前偏管理面板的单页界面，升级为更具 macOS 原生专业工具气质的桌面控制台。产品级目标分为两部分：

- 首页 `Monitor`：负责实时运行状态、健康度、趋势与异常监控
- 独立页 `Topology`：负责路由结构理解、流向分析与失败路径诊断

最终效果应接近 macOS 专业网络/运维工具，而不是普通 Web Admin 页面。

## 产品结构

### 1. Monitor

作为默认首页，承担以下职责：

- 展示网关、Provider、模型调用的实时状态
- 提供请求量、延迟、错误率等关键趋势
- 提供异常告警与运行摘要
- 成为产品第一视觉主舞台

### 2. Topology

作为独立页面，承担以下职责：

- 可视化 `Client / Gateway / Provider / Model` 路由结构
- 展示实时流量路径、默认链路与失败路径
- 支持节点与连线级别诊断
- 成为产品第二视觉主舞台

### 3. Resources / Settings

保留 Provider、Gateway、Logs、Rules、Advanced 等资源管理页面：

- 承担配置与维护职责
- 视觉语言与 Monitor / Topology 统一
- 不抢占监控首页与拓扑页的主视觉地位

## 首页 Monitor 设计

首页定位：`macOS 原生风格的 AI 路由监控驾驶舱`

### 信息架构

1. 顶栏工具区
- 页面标题与当前实例名
- 时间范围切换：`Live / 1h / 24h / 7d`
- 自动刷新状态
- 全局筛选与后续搜索入口

2. 关键指标总览
- `Running Gateways`
- `Active Providers`
- `Requests / min`
- `P95 Latency`
- `Error Rate`
- `Active Models`

3. 核心趋势与健康区
- 实时请求量与延迟趋势图
- 网络/Provider/Gateway 健康状态图

4. 运行明细区
- `Gateway Runtime Board`
- `Provider Health Board`
- `Top Models`
- `Recent Alerts`

5. 底部事件流
- 最近请求摘要
- fallback 事件
- 模型映射命中
- Gateway 生命周期事件

### 设计原则

- 第一眼 3 秒内判断系统是否健康
- 指标优先级高于图表，图表优先级高于明细列表
- 可视化服务于运维判断，不做无意义炫技
- 支持后续接入实时刷新与流式数据

## Topology 页面设计

页面定位：`macOS 原生风格的 AI 请求路由地图`

### 信息架构

1. 顶部控制栏
- `Live Flow / Static Topology / Failure Path`
- 时间范围
- Gateway / Provider / Model / Status 过滤器
- 搜索、缩放、居中、重置布局

2. 主画布
- 左到右固定四层：
  - `Clients / Entrypoints`
  - `Gateways`
  - `Providers`
  - `Models`

3. 详情面板
- 节点或连线详情
- 当前指标、上下游关系
- 默认模型、fallback、mapping rule
- 最近异常记录

### 可视化编码

- 线宽：请求量
- 线颜色：健康状态
- 线透明度：活跃程度
- 流动动画：实时方向
- 节点：信息型卡片节点，而非简单圆点

### 交互原则

- hover 高亮关联链路
- 点击节点或连线后打开详情
- 支持仅查看失败路径
- 保持结构清晰优先于动画炫目

## 视觉系统

整体视觉方向：`macOS Native + OLED Dark + Frosted Glass + Calm Operations`

### 背景

- 深灰蓝黑基底，不用纯黑
- 极轻的光晕与雾化层制造空间感
- 背景始终低噪音，为数据与路径让位

### 卡片

- 深色半透明磨砂卡片
- 极细描边
- 柔和阴影
- 主舞台卡、信息卡、辅助卡三层层级

### 侧边栏

- 采用 macOS source list 风格
- 选中态应更接近桌面原生工具，而非网页导航按钮
- 图标和文字统一偏轻量、克制

### 色彩

- 中性色作为大面积底色
- 彩色只用于状态与可视化
- 建议语义：
  - 蓝：基础信息/正常运行
  - 绿：健康/成功
  - 橙：警告
  - 红：错误
  - 紫：模型/智能路由

### 字体与数字

- 整体采用偏系统风格的无衬线字体栈
- 指标数字要更强、更稳定、更具桌面工具质感
- 标题层级保持清晰，不做夸张大字

### 动效

- 平滑、克制、低噪音
- 数字轻微过渡，图表平滑刷新，状态柔和切换
- 禁用夸张缩放、弹跳、闪烁与过度发光

## 首页与拓扑页的差异化

### Monitor
- 更卡片化
- 更偏状态与趋势
- 更适合快速扫描
- 节奏更丰富

### Topology
- 更画布化
- 背景更纯净、更深
- 结构与路径成为主角
- 更适合结构理解与链路诊断

## 设计优先级

### 第一阶段：整体气质
- 原生顶栏
- 原生侧边栏
- 深色桌面基底
- 卡片层级与 spacing 体系

### 第二阶段：首页监控能力
- 状态总览卡
- 趋势图
- 健康面板
- 事件流

### 第三阶段：拓扑页能力
- 节点系统
- 路径视觉编码
- 详情面板
- 诊断交互

## 非目标

- 不在本阶段引入复杂路由动画引擎
- 不在首页混入大面积拓扑画布
- 不把首页退化为普通配置表单页
- 不为了“像原生”而照抄系统设置应用排版

## 验收标准

1. 首页一眼可识别为监控驾驶舱而不是配置后台。
2. 拓扑页一眼可理解路由层级与主要链路。
3. 整体视觉明显更接近 macOS 专业桌面工具。
4. Monitor 与 Topology 的视觉语义统一但角色分明。
5. 后续实现时可按页面与组件逐步拆分推进。

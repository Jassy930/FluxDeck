## 阶段 33：macOS 原生壳 Sidebar 选择修复

- 问题表现：
  - 点击左侧 Sidebar 条目后，右侧 Detail 面板未切换。
- 根因：
  - `SidebarSection` 使用 `Identifiable(id: String)`，而 `List(selection:)` 绑定为 `SidebarSection?`，存在选择值类型推断不一致风险。
- 修复：
  - `SidebarSection` 改为 `Hashable`（移除 `Identifiable`）。
  - Sidebar 列表改为 `List(..., id: \\.self, selection: $selectedSection)`，显式以枚举值本身作为选择键。
- 验证结果：
  - `xcodebuild test -project apps/desktop-macos-native/FluxDeckNative.xcodeproj -scheme FluxDeckNative -quiet` 通过（5 tests passed）

## 阶段 34：Logs 面板过滤切换抖动修复

- 问题表现：
  - 切换 `Only Errors` 开关时，Logs 面板内容区域出现上下跳动。
- 根因：
  - 过滤结果为空时，视图在 `List` 与 `EmptyStateView` 之间切换，叠加 `List` 的隐式更新动画，导致布局抖动。
- 修复：
  - Logs 面板改为固定容器（`ZStack` + 全高 frame），始终保留 `List` 容器。
  - 空态与加载态改为覆盖层显示，避免主体结构切换。
  - 对 Logs 容器显式关闭隐式动画（`transaction.animation = nil`）。
- 验证结果：
  - `xcodebuild test -project apps/desktop-macos-native/FluxDeckNative.xcodeproj -scheme FluxDeckNative -quiet` 通过（5 tests passed）

## 阶段 35：Settings 面板顶对齐修复

- 问题表现：
  - 切换到 Settings 后内容区域视觉上居中，不与其它面板保持“顶部起始”一致。
